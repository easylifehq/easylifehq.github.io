# Operating Mode

Project: EasyLife

Mode: demo-forge

Label: Personal Assistant Product Studio

Lead reviewer: Simon and Robin

## Done Standard

Feels like a calm, useful personal assistant app: one daily start point, connected modules, concrete copy, mobile comfort, and no feature dump.

## Planning Rules

Start with the signed-in user's first daily action, then shape module screens around what the user can do next. Prefer connected suite hierarchy, progressive disclosure, and subtraction before adding more surfaces.

## First Screen Contract

The protected first screen must show one clear daily next action, today context, and compact module status before secondary navigation, product explanation, feature inventories, or decorative chrome.

## Forbidden Patterns

No generic SaaS command-center framing, no all-in-one dashboard dump, no visible internal build words, no fake backend/AI claims, no auth/Firebase/deploy/package changes, and no broad rewrites.

## Required Gates

Simon visual/taste review, Robin concrete copy review, visual screenshot check, information staging gate, launch gate, security guardrails, and source-of-truth build.

## Mode Reminder

The phase system still applies. This mode tells the fleet to judge EasyLife like a personal assistant product, not a generic dashboard or marketing site. Do not replace build checks, guardrails, staging, review, or repair gates with taste alone.
