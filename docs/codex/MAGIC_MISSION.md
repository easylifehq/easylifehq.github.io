# Magic Mission

## 12-Hour Outcome

EasyLife should feel like a calm personal operating system with a clear shell, reliable navigation, and useful first-pass workflows across list, notes, calendar, workout, and settings screens.

## Target User

The target user is a busy solo operator who wants fewer scattered tools and a more peaceful way to track tasks, notes, schedule, habits, and personal routines.

## Product Promise

EasyLife helps people see what matters today, capture small details quickly, and move between life areas without feeling like they are configuring enterprise software.

## Visual Direction

Simon should keep the interface quiet, tactile, modern, and practical: strong hierarchy, mobile comfort, visible state, restrained motion, and no generic SaaS hero clutter.

## Non-Goals

- Do not edit secrets, auth, payments, Firebase rules, backend config, package files, generated build output, deployment config, or production data.
- Do not invent real integrations, pricing, accounts, analytics, or customer claims.
- Do not redesign every screen in one task; prefer coherent screen-level or workflow-level slices.

## Definition Of Magic

- The first screen makes the product purpose obvious.
- Core routes feel visually related instead of patched together.
- Mobile layouts have no obvious overflow, clipped text, or cramped controls.
- Simon's next review names fewer generic or unfinished areas.
- Completed tasks build toward one recognizable product direction.
