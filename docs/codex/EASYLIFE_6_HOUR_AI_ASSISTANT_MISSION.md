# EasyLife 6 Hour AI Assistant Mission

## Goal

Turn EasyLife into one slick personal assistant app: Today is the command surface, Capture is the inbox, Plan is the day timeline, Notes are memory, and More holds optional modules quietly.

## Product Bet

The assistant should feel intelligent before it has real model access. Build the interface, local command grammar, suggested next actions, and review loops first. Real AI/API work stops until backend, secrets, auth, privacy, and model policy are intentionally designed.

## Creative Direction

- Techy, calm, precise, dark-friendly, compact.
- One primary next action per screen.
- Fewer cards, fewer labels, fewer module names.
- Assistant language should be direct and operational, not demo/presentation copy.
- Combine surfaces through shared command, capture, plan, and memory concepts instead of adding another dashboard.

## Six Hour Shape

Batch size stays 1. Each task should make one visible or proof change, build, and leave a clean checkpoint.

1. Repair the current EasyList/Capture quarantine.
2. Make Today feel like an AI command surface.
3. Add local command intent examples without real API calls.
4. Tie capture items to plan/memory language.
5. Tighten Calendar/Plan into a day timeline.
6. Tighten Notes into assistant memory.
7. Make More an optional module drawer instead of a product grid.
8. Polish mobile first impression.
9. Run copy and proof passes.

## Stop Conditions

Stop instead of improvising if a task needs backend behavior, auth changes, secrets, payments, Firebase rules/config, package/dependency changes, generated output, deployment config, stored data migration, or a real AI provider integration.
